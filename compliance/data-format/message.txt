1.  Conventions and Definitions

   The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT",
   "SHOULD", "SHOULD NOT", "RECOMMENDED", "NOT RECOMMENDED", "MAY", and
   "OPTIONAL" in this document are to be interpreted as described in BCP
   14 [RFC2119] [RFC8174] when, and only when, they appear in all
   capitals, as shown here

2.  Message

2.1.  Version

   0.2.0

   Note: This is the version of the message specification, not the
   message format.

2.2.  Overview

   A message is a formatted structure that contains encrypted data and
   associated metadata.

2.3.  Structure

   A message is a sequence of bytes that is the serialization of the
   following, in order:

   *  Message Header (message-header.md)

   *  Message Body (message-body.md)

   If the message header (message-header.md) contains an algorithm suite
   (../framework/algorithm-suites.md) in the algorithm suite ID
   (message-header.md#algorithm-suite-id) field that contains a
   signature algorithm (../framework/algorithm-suites.md#signature-
   algorithm), the message MUST also contain a message footer (message-
   footer.md), serialized after the message body (message-body.md).
